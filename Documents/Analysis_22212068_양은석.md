## <a id="intro"></a>1. Introduction

현대 사회에서 주거지를 선택할 때 많은 사람들은 주변 환경, 치안, 대기질, 부동산 시세 등 다양한 요소를 고려해야 하지만, 이러한 정보는 여러 기관과 플랫폼에 흩어져 있어 한눈에 파악하기 어렵다. 또한 개인의 경험이나 주변 의견에 의존한 주관적인 판단이 개입되기 쉬워, 객관적인 기준에 기반한 합리적인 의사결정이 어려운 경우가 많다. 이로 인해 사용자는 자신에게 적합한 ‘살기 좋은 동네’를 찾는 과정에서 시간적·인지적 부담을 겪고 있다.

이러한 문제를 해결하기 위해 공공데이터를 통합·분석하여 제공하는 ‘OpenScore-Housing(가칭: 데이터 기반 주거지 가치 평가 시스템)’을 제안한다. 본 시스템은 범죄율, 대기질, 부동산 시세 등 다양한 객관적 지표를 종합적으로 반영하여 지역별 점수를 산출하고, 사용자 맞춤형 기준에 따라 주거지를 비교·탐색할 수 있도록 지원한다. 이를 통해 사용자는 데이터 기반의 합리적인 판단을 통해 자신에게 가장 적합한 주거 환경을 보다 쉽게 찾을 수 있다.

해당 문서는 Conceptualization Document에 이은 두 번째 단계인 Analysis 단계의 문서로, Use Case 및 Domain 분석을 수행하고, 시스템의 전반적인 User Interface 구조와 사용자 상호작용 방식을 소개하는 내용을 다루고 있다.

## <a id="domain"></a>3. Domain analysis

1) User
사용자 클래스다. 사용자가 설정한 지역이나 가중치에 대한 값을 포함한다

2) Interface
인터페이스 클래스다. 사용자에게 표시할 UI에 관련된 클래스다

3) API
API클래스다. 공공 데이터 api가 제공하는 데이터를 가져오는 클래스이다.

4) Crawling
크롤링 클래스다. api를 제공하지 않는 공공 데이터를 수집하는 클래스이다

5) Scheduler
스케줄러 클래스다. 자동으로 데이터를 갱신하는 클래스다

6) DB
DB클래스다. 각종 데이터를 저장하는데 사용하는 클래스다

7) Error send
에러 발송 클래스다. 에러가 발생하면 관리자에게 에러를 발송하는 클래스다

8) map score
맵 점수 클래스다. 각 데이터를 기반으로 가중치에 활용한 점수를 보관하는 클래스다

9) variable
변수 클래스다. 맵 점수를 제외한 각종 변수를 저장하는 클래스다.
